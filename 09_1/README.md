# Лабораторная работа
## Исправления по файлам

### `Task01.cpp` (рюкзак по весу и налогу)

1. **Портативность:** удалены `#include <Windows.h>` и вызовы `SetConsoleCP` / `SetConsoleOutputCP` — на macOS/Linux заголовок отсутствует, для чтения из файла кодовая страница консоли не нужна.
2. **Макрос `INT_MAX`:** добавлен `#include <climits>`; раньше использовался `INT_MAX` без гарантированного объявления.
3. **Проверка `N` и `Z`:** условие было `N < 0 || Z < 0`, из‑за чего пропускались нули при формулировке «должны быть больше 0»; заменено на `N <= 0 || Z <= 0`.
4. **Стиль и читаемость:** пробелы вокруг операторов, единообразные фигурные скобки, константа переименована в `kInf` (вместо `INF` в стиле макроса).
5. **Опечатка в выводе:** в строке вывода латинская «C» в слове «Суммарный» заменена на кириллическую «С» (рядом в коде оставлен комментарий `FIX_ME`).
6. **Входные данные:** в папке создан пример **`input.txt`** (первая строка `N Z`, далее веса и налоги).
7. **Совместимость со старым C++:** объявление `vector<vector<bool>>` заменено на `vector<vector<bool> >` (пробел между закрывающими `>`). При сборке без `-std=c++11` и новее компилятор воспринимает `>>` как битовый сдвиг и выдаёт ошибку *«a space is required between consecutive right angle brackets»*.

### `Task02.cpp` (черепашка, минимальная сумма)

1. **Надёжность ввода-вывода:** добавлены проверки открытия `a.txt` и создания `b.txt` с сообщением в `cerr` и кодом выхода `1`.
2. **Ограничение на `N`:** после чтения `N` добавлена проверка `1 < N < 80` из условия, чтобы не создавать доску нулевого или заведомо неверного размера.
3. **`main`:** добавлен явный `return 0` в конце (раньше функция завершалась без возврата).
4. **Текст условия в комментарии:** исправлена опечатка «по пасть» → «попасть», выровнены пробелы после точек.
5. **Входные данные:** в папке создан пример **`a.txt`** для проверки (доска `2×2`); программа записывает ответ в **`b.txt`**.
6. **Совместимость со старым C++:** вложенные векторы записаны как `vector<vector<int> >` вместо `vector<vector<int>>` по той же причине, что и в `Task01.cpp` (см. пункт 7 там).

### `Task03.cpp` (K-ичные числа с двумя нулями подряд)

1. **`setlocale`:** добавлен `#include <clocale>`; локаль в `main` задана как `ru_RU.UTF-8` (удобнее для UTF-8 в терминале на macOS/Linux).
2. **Имена переменных:** вместо `_nz`, `_oz`, `_tz` используются `prev_nz`, `prev_oz`, `prev_tz` (имена с ведущим подчёркиванием в глобальной области зарезервированы стандартом; в функции допустимы, но стиль ухудшает читаемость).
3. **Ограничения из условия:** добавлена проверка `2 ≤ K ≤ 10`, `1 < N < 20`, `N + K < 26` с сообщением об ошибке.
4. **Тип результата:** возвращаемое значение приводится к `double` через `static_cast`, чтобы явно соответствовать формулировке «вещественное число».
5. **Лишний заголовок:** удалён неиспользуемый `#include <cmath>`.

### `Task04.cpp` (разбор логического выражения)

1. **Требование «без оператора цикла»:** функция `skipWhitespaces` переписана с `while` на **рекурсию** (в условии запрещены циклы).
2. **Предупреждения `-Wsign-compare`:** индекс позиции заменён с `int` на **`size_t`**, сравнения идут с `expression.size()`.
3. **`setlocale`:** убран из `evaluateExpression` (вызывался на каждом шаге разбора); оставлен один вызов в `main`.
4. **Разбор префиксов:** вместо `substr` для `And`/`Or` используется `compare(pos, len, ...)`, без лишних аллокаций строк.
5. **Код возврата при ошибке:** вместо `-1` используется `1` (положительный код ошибки — более привычная практика для `main`).
6. **Приведение для `isspace`:** символ приводится к `unsigned char` перед вызовом `isspace`, чтобы избежать неопределённого поведения для отрицательных `char`.
