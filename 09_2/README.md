# Лабораторная работа 09_2
## Исправления по файлам

### `Task01.cpp` (стек на связанных узлах)

1. **Деструктор `~Stack()`:** отсутствовал — при любом завершении программы все узлы стека оставались в памяти (утечка памяти); добавлен деструктор с обходом и удалением всех узлов.
2. **`print()` и `getTop()`:** добавлен квалификатор `const`, так как методы не изменяют состояние объекта.
3. **Тривиальные комментарии:** удалены очевидные построчные комментарии (`// Новый узел указывает на текущую вершину`, `// Удаляем узел` и т. д.) — они дублировали код, не добавляя смысла.

### `Task02.cpp` (две очереди из 10 чисел)

1. **`#include <windows.h>`:** заголовок недоступен на macOS/Linux; удалён.
2. **`setlocale(LC_ALL, "Russian")`:** Windows-специфичная строка локали, на macOS возвращает ошибку без эффекта; удалена (UTF-8 поддерживается терминалом по умолчанию).
3. **Отсутствие конструктора:** вместо конструктора использовался отдельный метод `Inicializaciya()` — не C++-идиома, поля не инициализировались автоматически при создании объекта; заменён нормальным конструктором с инициализирующим списком.
4. **Деструктор `~Queue()`:** отсутствовал — утечка памяти; добавлен с полным обходом и удалением узлов.
5. **Стиль именования:** имена в русской транслитерации (`Uzel`, `Ochered`, `Nachalo`, `Konec`, `Znachenie`, `Sleduyushiy`, `Noviy`, `Vremenny`, `Tekushiy`) нарушали единый стиль проекта; переименованы в английские эквиваленты (`Node`, `Queue`, `head`, `tail`, `data`, `next`, `newNode`, `temp`, `current`).
6. **Код возврата при ошибке ввода:** был `return 0` — неверный код; заменён на `return 1` (код ошибки).

### `Task03.cpp` (упорядоченный список из файла)

1. **Деструктор `~LinkedList()`:** отсутствовал — утечка памяти; добавлен с полным обходом и удалением узлов.
2. **Дублирующая `friend`-функция:** свободная функция `readFromFile(LinkedList&, const string&)` просто вызывала одноимённый метод класса без какой-либо собственной логики — избыточная обёртка удалена вместе с объявлением `friend`.
3. **Файл `task3_input.txt`:** создан пример входного файла (`7` чисел) для проверки работы программы.

### `Task04.cpp` (циклический двусвязный список, удаление элементов)

1. **Кодировка:** файл был в Windows-1251 — заново записан в UTF-8; весь русскоязычный текст восстановлен корректно.
2. **`#include <iostream>` и `using namespace std;`:** отсутствовали — файл не компилировался.
3. **`bool deletedSomething = false;`:** переменная объявлялась, но нигде не читалась и не использовалась; удалена.
4. **`head` в `public`:** прямой доступ к внутреннему указателю нарушал инкапсуляцию; поле перенесено в `private`, приватным стал и вспомогательный метод `deleteNode()`.
5. **Деструктор `~DoubleList()`:** на первой итерации удалял `head` (через `tmp`), после чего сравнивал `curr == head` с уже освобождённым указателем — неопределённое поведение; исправлено: цикл разрывается (`tail->next = nullptr`) и список обходится линейно.
6. **`printList()` без проверки пустого списка:** при вызове на пустом списке (head == nullptr) немедленно разыменовывался нулевой указатель → crash; добавлена проверка с выводом `(пустой список)`.
7. **`tail->data` без проверки на `nullptr`:** `removeSides()` может вернуть `nullptr` (пустой список после обработки), и прямое обращение `tail->data` приводило к crash; добавлена проверка.
8. **`printList()` — переход с `while (true)` на `do-while`:** логика прохода по циклическому списку стала явной.

---

### `Task05.cpp` (STL `list`, перемещение среднего элемента через `splice`)

1. **`setlocale(LC_ALL, "Russian")`:** Windows-специфичная строка локали; удалена.
2. **`Print(const list<int> L)`:** принимала контейнер по значению — лишнее копирование всего списка при каждом вызове; заменена на `const list<int>&`.
3. **Порядок валидации `n1`:** проверка `n1 % 2 != 1` выполнялась **после** вызова `Input(L1, n1)` — пользователь уже вводил данные, получая сообщение об ошибке постфактум; проверка перенесена **до** вызова `Input()`.
4. **`it++` → `++it`:** в цикле `Print` использовался постфиксный инкремент итератора — заменён на префиксный `++it` (нет лишней копии).
5. **`time(0)` → `time(nullptr)`:** более идиоматичная современная запись.
6. **`srand` приведение типа:** добавлен `static_cast<unsigned>` для явного преобразования возвращаемого `time()`.
7. **`#include <cstdlib>`:** добавлен для явного объявления `srand` и `rand`.
8. **Файл `a.txt`:** создан пример файла с 14 числами для тестирования режима чтения из файла.
